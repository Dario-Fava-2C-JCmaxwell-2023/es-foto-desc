🔍 v1.0 — Documentazione ufficiale

Descrittore Foto AI

Un'applicazione web real-time che usa l'intelligenza artificiale multimodale per analizzare qualsiasi immagine e restituire una descrizione dettagliata in italiano. Dopo l'analisi, l'utente può continuare la conversazione con l'AI tramite una chat contestuale.

Node.js + WebSocket
Groq API · Llama 4 Scout
Porta predefinita: 3000
JavaScript puro (niente framework)

🧩 Stack tecnologico

Il progetto è interamente scritto in JavaScript, senza framework aggiuntivi né bundler. Le dipendenze esterne sono ridotte al minimo.

🟢
Node.js (runtime)
Esegue il server HTTP e WebSocket lato backend, senza browser.
📡
ws (WebSocket)
Libreria leggera per comunicazione bidirezionale in tempo reale tra browser e server.
🔐
dotenv
Carica le variabili d'ambiente dal file .env senza esporle nel codice.
🤖

Servizio cloud ultra-veloce per inferenza LLM. Usato per l'analisi visiva.
🦙
Llama 4 Scout
Modello multimodale di Meta. Legge testo e immagini nello stesso contesto.
📝
marked.js (CDN)
Parser Markdown lato client. Trasforma le risposte AI in HTML formattato.

🔑 Configurazione .env

Il file .env nella root del progetto contiene le variabili d'ambiente necessarie. Non condividere mai questo file pubblicamente: contiene la tua chiave API privata. 

# File: .env

# Chiave API Groq (gratuita, senza carta di credito)
# Ottienila su: https://console.groq.com/keys
GROQ_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

# Porta su cui il server ascolterà (opzionale, default: 3000)
PORT=3000.ENV
Variabile Obbligatoria Default Descrizione
GROQ_KEY ✅ Sì — Chiave segreta per autenticarsi all'API Groq. Il server non si avvia senza di essa.
PORT ❌ No 3000 Porta TCP su cui server HTTP e WebSocket ascoltano contemporaneamente.
⚠️

Attenzione: se GROQ_KEY manca o contiene il valore segnaposto inserisci_la_tua_chiave_qui, il server terminerà immediatamente con un messaggio di errore.

Come ottenere una chiave Groq gratuita:

  1. Vai su console.groq.com/keys
  2. Crea un account (nessuna carta di credito richiesta)
  3. Clicca "Create API Key" e copia il valore
  4. Incollalo nel file .env al posto del valore di esempio

Riconnessione automatica

Se la connessione WebSocket cade (es. riavvio del server), il client attende 3 secondi e tenta automaticamente la riconnessione, aggiornando l'indicatore di stato nel badge in cima alla pagina.

📦 Compressione immagini (Canvas API)

Prima di inviare l'immagine al server, il browser la normalizza e comprime usando la Canvas API. Questo risolve problemi di compatibilità con formati come AVIF e file troppo grandi.

Algoritmo di ridimensionamento

const MAX = 1024; // lato lungo massimo in pixel
let w = img.width, h = img.height;

if (w > MAX || h > MAX) {
    if (w > h) { h = Math.round(h * MAX / w); w = MAX; }
    else       { w = Math.round(w * MAX / h); h = MAX; }
}

// Converte in JPEG con qualità 85%
const datiBase64 = canvas.toDataURL("image/jpeg", 0.85);JS
📐
Lato lungo max 1024px
Le proporzioni originali vengono sempre mantenute.
🎨
Output JPEG al 85%
Buon compromesso tra qualità visiva e dimensione del payload.
🔄
Formato normalizzato
AVIF, HEIC, WebP e qualsiasi formato supportato dal browser vengono tutti convertiti in JPEG.

🌐 Server HTTP

Il server HTTP è creato con il modulo nativo http di Node.js (senza Express). Serve i file statici dalla cartella public/. 

const serverHttp = http.createServer((richiesta, risposta) => {
    // "/" → "/index.html"
    const url = richiesta.url === "/" ? "/index.html" : richiesta.url;
    const percorsoFile = path.join(__dirname, "public", url);

    fs.readFile(percorsoFile, (errore, contenuto) => {
        if (errore) { risposta.writeHead(404); return; }
        // Header no-cache: il browser ricarica sempre l'ultima versione
        risposta.writeHead(200, { "Cache-Control": "no-store, no-cache" });
        risposta.end(contenuto);
    });
});JS
💡

Gli header no-store, no-cache sono impostati deliberatamente per evitare che il browser mostri versioni obsolete di index.html durante lo sviluppo.

📡 Server WebSocket

Il server WebSocket condivide la stessa porta del server HTTP grazie all'opzione { server: serverHttp }. Gestisce un client alla volta per connessione. 

const serverWs = new WebSocketServer({ server: serverHttp });

serverWs.on("connection", (connessione) => {

    connessione.on("message", async (messaggioGrezzo) => {
        const msg = JSON.parse(messaggioGrezzo.toString());

        if (msg.tipo === "analizza_foto") {
            // → chiama Groq, manda indietro la descrizione
        } else if (msg.tipo === "domanda_chat") {
            // → ricostruisce contesto, chiama Groq, manda risposta
        }
    });

    connessione.on("close", () => console.log("👋 Browser disconnesso"));
});JS

⚙️ Funzioni di supporto

chiamaGroq(messages)

Effettua una chiamata POST all'API Groq e restituisce il testo della risposta del modello. Lancia un'eccezione se la risposta HTTP non è 2xx.

async function chiamaGroq(messages) {
    const risposta = await fetch(URL_GROQ, {
        method: "POST",
        headers: {
            "Authorization": `Bearer ${GROQ_KEY}`,
            "Content-Type": "application/json"
        },
        body: JSON.stringify({ model: MODELLO, messages, max_tokens: 1024 })
    });
    if (!risposta.ok) throw new Error(...);
    const dati = await risposta.json();
    return dati.choices[0].message.content;
}JS

invia(connessione, dati)

Serializza un oggetto in JSON e lo invia sulla connessione WebSocket. Prima controlla che la connessione sia ancora aperta (readyState === OPEN) per evitare errori su connessioni già chiuse.

function invia(connessione, dati) {
    if (connessione.readyState === connessione.OPEN) {
        connessione.send(JSON.stringify(dati));
    }
}JS

escapeHtml(testo) — Frontend

Sostituisce i caratteri speciali HTML (&, <, >) con le entità equivalenti prima di inserire il testo dell'utente nel DOM. Previene attacchi XSS.

function escapeHtml(testo) {
    return testo
        .replace(/&/g,  "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;");
}JS

🛑 Gestione degli errori

🔑
Chiave Groq mancante
Il server esce immediatamente con process.exit(1) e stampa un messaggio di errore chiaro in console.
🌐
Errore API Groq
Se lo status HTTP non è 2xx, viene lanciata un'eccezione con il codice di stato e il corpo della risposta.
📁
File non trovato
Il server HTTP risponde con 404 se un file non esiste nella cartella public/.
🖼️
Immagine non leggibile
Se il Canvas API non riesce a caricare l'immagine, viene mostrato un alert all'utente e il pulsante viene riabilitato.
📡
WebSocket disconnesso
L'indicatore diventa rosso e il client riprova ogni 3 secondi fino alla riconnessione.
⛔
Errori runtime server
Qualsiasi eccezione non gestita viene intercettata da un try/catch e inviata al browser come messaggio di tipo errore.

🛡️ Note sulla sicurezza

⚠️

Uso locale: questo progetto è progettato per uso locale o su rete privata. Non esporre il server su Internet pubblico senza aggiungere autenticazione.

🔴

Proteggi il .env: il file .env contiene la tua chiave API privata. Non includerlo mai in commit Git (è già in .gitignore) e non condividerlo.

✅

XSS prevenuto: i messaggi dell'utente nella chat vengono sempre sanificati tramite escapeHtml() prima di essere inseriti nel DOM.

✅

Nessun dato persistito: le immagini e le conversazioni non vengono mai scritte su disco dal server. Tutto rimane in memoria per la durata della connessione.

💡

Chiave API lato server: la GROQ_KEY non è mai esposta al browser. Solo il server Node.js la conosce e la usa per chiamare l'API.
